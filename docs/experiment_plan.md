# 基于 MRI 影像的 SVM 疾病分类实验方案

## 1. 题目建议

推荐主命题：

**基于软间隔核支持向量机的脑 MRI 影像 AD 与 NORMAL 分类研究**

扩展命题：

1. 基于 PCA-SVM 的脑 MRI 影像 AD 与 NORMAL 分类及核函数比较
2. 基于软间隔 SVM 的 MCI 与 NORMAL 分类研究
3. 基于多类 SVM 的 AD/MCI/NORMAL 三分类研究

本方案默认先完成主命题，再按同一套代码扩展到其余命题。

## 2. 与讲义的对应关系

本实验直接对应 `Lecture_Note/chapter6_lecture_notes.pdf` 中的以下内容：

1. `6.1` 间隔与支持向量：解释为什么 SVM 追求最大间隔分类面。
2. `6.2` 对偶问题与 KKT：说明训练后模型主要由支持向量决定。
3. `6.3` 核函数：比较线性核与 RBF 核的分类效果。
4. `6.4` 软间隔与正则化：通过惩罚参数 `C` 控制间隔与误分类的平衡。
5. `6.6` 核方法：将线性分类器通过核技巧扩展到非线性场景。

## 3. 数据概况

数据目录为 `processed/`，文件格式均为 `.nii.gz`，文件名中含有：

1. 受试者编号，如 `002_S_0619`
2. 类别标签，如 `AD`、`MCI`、`NORMAL`
3. 随访时间点，如 `m00`、`m06`、`m12`、`m24`

当前数据统计如下：

1. `AD`：577 个影像，172 名受试者
2. `MCI`：1598 个影像，380 名受试者
3. `NORMAL`：814 个影像，218 名受试者

适合作为主实验的基线子集：

1. 仅取 `m00` 作为基线时点
2. 先做 `AD vs NORMAL` 二分类
3. 再扩展到 `MCI vs NORMAL`、`AD vs MCI` 和三分类

这样可以避免同一受试者多个时间点同时进入训练集和测试集造成数据泄漏。

## 4. 研究目标

### 4.1 主目标

利用 MRI 影像训练 SVM 分类器，区分 `AD` 与 `NORMAL` 个体。

### 4.2 次目标

1. 比较线性核与 RBF 核的效果差异
2. 分析软间隔参数 `C` 对泛化性能的影响
3. 比较不同任务难度：`AD vs NORMAL`、`MCI vs NORMAL`、`AD vs MCI`
4. 在多分类场景下评估 SVM 的整体可行性

## 5. 实验假设

1. `AD vs NORMAL` 的可分性最好，分类性能应优于 `MCI vs NORMAL`
2. 对 MRI 这种高维且非完全线性可分的数据，RBF 核通常优于线性核
3. `PCA + SVM` 比“直接高维体素输入”更稳定，且更适合课程型实验
4. 采用受试者级划分和分层交叉验证后，实验结果更可信

## 6. 方法设计

### 6.1 样本选择

主实验默认使用以下设置：

1. 时间点：`m00`
2. 主任务：`AD` 与 `NORMAL`
3. 扩展任务：
   - `MCI` 与 `NORMAL`
   - `AD` 与 `MCI`
   - `AD/MCI/NORMAL` 三分类

### 6.2 数据预处理

考虑到影像已处于统一空间，本实验采用轻量但稳定的预处理：

1. 读取 `.nii.gz` 三维 MRI 数据
2. 使用非零体素估计强度分位数
3. 对每个样本做强度裁剪，降低极端值影响
4. 对每个样本做 z-score 标准化
5. 将体积下采样到统一的小尺寸，例如 `32 x 32 x 32`
6. 展平成一维特征向量

### 6.3 特征工程

为了让 SVM 在高维 MRI 上更稳定，使用如下策略：

1. 先做图像级轻量标准化
2. 再在机器学习流水线中使用 `StandardScaler`
3. 使用 `PCA` 将特征降至固定维度，默认 100 维

原因：

1. 降低维度灾难和过拟合风险
2. 缩短训练时间
3. 更符合课程作业场景

### 6.4 模型设置

比较两类模型：

1. 线性 SVM
2. RBF 核 SVM

统一采用软间隔设置，并通过网格搜索调参：

1. `C`：`[0.1, 1, 10]`
2. `gamma`：仅对 RBF 核搜索，默认 `["scale", 0.1, 0.01]`

### 6.5 划分与验证策略

为了避免数据泄漏，采用**受试者级划分**：

1. 先按受试者编号分层切分训练集和测试集
2. 训练集内部再使用 `StratifiedGroupKFold` 做交叉验证
3. 交叉验证时仍按受试者分组

默认参数：

1. 测试集比例：20%
2. 交叉验证折数：5 折
3. 随机种子：42

## 9. 风险控制

### 9.1 数据泄漏

不能把同一受试者的多个时间点同时随机分到训练集和测试集。

### 9.2 类别不平衡

`AD`、`MCI`、`NORMAL` 数量不同，因此 SVM 使用 `class_weight="balanced"`。

### 9.3 高维过拟合

通过下采样、标准化、PCA、交叉验证共同缓解。
